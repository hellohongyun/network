### ③ 机场精准分流（核心改造）

**你的方案实现了**，并且我加了一个额外优化：

```
省流版地区组（无★）= C → B → A 故障转移
稳定版地区组（带★）= A → B → C 故障转移
```

底层结构：
```
[C] 新加坡 = url-test，只抓 AirportC 的新加坡节点
[B] 新加坡 = url-test，只抓 AirportB 的新加坡节点
[A] 新加坡 = url-test，只抓 AirportA 的新加坡节点
     ↓
🇸🇬 新加坡  = fallback [ [C]新加坡 → [B]新加坡 → [A]新加坡 ]  ← 视频/通用
🇸🇬 新加坡★ = fallback [ [A]新加坡 → [B]新加坡 → [C]新加坡 ]  ← AI专用
```

对应应用组引用关系：

| 类型 | 引用的地区组 | 逻辑 |
|------|------------|------|
| 📹 YouTube / 🎥 Netflix / 🎧 Spotify | `🇸🇬 新加坡`（省流版）| C先用，挂了换B，再挂换A |
| 🤖 Claude / ChatGPT / Gemini | `🇸🇬 新加坡★`（稳定版）| A先用，挂了换B，再挂换C |
| 🎮 Steam | `DIRECT` 置顶 | 完全不消耗机场流量 |

---

### ⚠️ 一个需要注意的地方

如果某个机场**没有某个地区的节点**（比如机场C没有台湾节点），`[C] 台湾` 这个子组会是空的，mihomo 会把它标为不可用，然后 fallback 自动跳到 `[B] 台湾`。这是正确的行为，不会报错，但建议在面板里检查一下各子组是否有节点加载进来。
