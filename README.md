**一台路由器，四个 WiFi，四种网络人格。**

---

## 它是什么？
这是一套运行在 OpenWrt 路由器上的 Mihomo（Clash Meta）配置方案。它把你家的网络变成一个**智能交通枢纽**——不同的 WiFi 连接，不同的网络策略，全自动运行，零感知切换。

## 核心亮点
### 1. 连不同 WiFi = 不同网络身份
| 连接的 WiFi | 你获得的网络 | 适合场景 |
| --- | --- | --- |
| **ap-direct** | 裸连直链，零延迟 | 国内购物、视频、游戏 |
| **ap-rule** | 智能分流，该直连直连，该代理代理 | 日常办公、开发、社交 |
| **ap-Global** | 全球畅通，所有流量走代理 | 需要完全翻墙的场景 |
| **ap-34x** | 美国住宅 IP，和当地人一样 | 需要纯净 IP 的特殊场景 |


手机想直连？连 ap-direct。  
电脑想分流？连 ap-rule。  
不用装任何客户端，不用改任何设置，切 WiFi 就行。

### 2. 24 个平台精准控制
每个平台都有独立的策略组，你可以精确到 **"Claude 走日本，ChatGPT 走美国，YouTube 走新加坡"** 这种级别：

| 类别 | 平台 |
| --- | --- |
| AI | Claude / ChatGPT / Gemini / DeepSeek |
| 视频 | YouTube / Netflix / TikTok / Spotify / Disney+ |
| 社交 | Telegram / X / Instagram / Facebook / Discord |
| 开发 | Google / GitHub / Cloudflare / Figma / Notion |
| 系统 | Microsoft / Apple |
| 金融 | 加密货币 / PayPal |
| 游戏 | Steam |


没有覆盖到的平台？别担心：

+ 海外 AI 平台 → AI 兜底规则自动捕获
+ 其他海外网站 → `geolocation-!cn` 自动代理
+ 国内网站 → 自动直连

### 3. 三机场智能调度
同时接入 3 个机场（或更多），系统自动帮你**用对的机场做对的事**：

```plain
看 YouTube → 最便宜的机场先上（省钱）
    ↓ 掉线了
    自动切换到第二便宜的
    ↓ 也掉线了
    自动切换到最好的

用 Claude → 最好的机场先上（稳定）
    ↓ 掉线了
    自动切换到第二好的
    ↓ 也掉线了
    自动切换到最便宜的
```

你选地区，系统选机场。**视频省钱，AI 稳定，全自动。**

### 4. 9 个地区自由切换
| 主要地区 | 特性 |
| --- | --- |
| 🇭🇰 香港 / 🇯🇵 日本 / 🇸🇬 新加坡 / 🇺🇸 美国 / 🇹🇼 台湾 | 三机场独立测速 + 故障转移 |


| 次要地区 | 特性 |
| --- | --- |
| 🇬🇧 英国 / 🇩🇪 德国 / 🇫🇷 法国 / 🇰🇷 韩国 | 全机场联合测速 |


每个主要地区背后是 **3 个机场的独立测速组**，自动选各机场最快的节点。你在面板上看到的是简洁的国旗，背后是精密的调度引擎。

### 5. DNS 防泄漏
+ **fake-ip 模式**：域名不会暴露给本地 DNS
+ **国内外分流解析**：国内域名走阿里/腾讯 DOH，海外域名走 Google/Cloudflare DOH
+ **ECS 优化**：海外 DNS 查询附带国内子网提示，获得更优的 CDN 调度
+ **respect-rules**：连 DNS 查询本身也遵守代理规则

### 6. 一键扩展
要加新平台？**3 行搞定：**

1. 策略组加一个
2. 规则加一条
3. 规则集加一个

要加新网段？**也是 3 步：**

1. 住宅 IP 节点加一个
2. 策略组加一个
3. 网段路由加一行

## 快速开始
### 第一步：填写订阅链接
打开 `configs/v4.yaml`，找到第六节"机场订阅"，填入你的机场订阅 URL：

```yaml
AirportA:
  url: "你的最好机场订阅链接"

AirportB:
  url: "你的次要机场订阅链接"

AirportC:
  url: "你的最便宜机场订阅链接"
```

### 第二步：上传到路由器
将 `configs/v4.yaml` 上传到 OpenWrt 的 Mihomo 插件配置目录，在插件页面选择该配置文件并启用。

### 第三步：打开面板
浏览器访问 `http://路由器IP:9090/ui`，你会看到：

+ **第一排**：出口组——控制各网段的出口
+ **第二排**：应用策略组——每个平台选一个地区
+ **第三排**：地区组——通常不需要动
+ **第四排**：机场子组——基础设施，几乎不碰

### 第四步：连上 WiFi
把手机/电脑连到对应的 WiFi AP，就这样。

## 日常操作指南
### 切换 Claude 的出口地区
1. 打开面板 → 找到 `🤖 Claude 32.x`
2. 从 `🇯🇵 日本★` 切换到 `🇺🇸 美国★`
3. 完成。Claude 现在走美国（最好机场优先）

### 看 YouTube 省流量
`📹 YouTube 32.x` 默认已经是省流版地区组（C→B→A），你只需要选地区就行。系统会自动优先使用最便宜的机场。

### Steam 下载游戏
`🎮 Steam 32.x` 默认 DIRECT，游戏下载不消耗机场流量。如果需要访问外区商店，手动切换到对应地区。

### 所有流量都想走代理
把设备连到 **ap-Global** WiFi 即可。不需要在面板做任何操作。

### 排查某网站不通
1. 打开面板，查看该网站命中了哪条规则（Connections 页面）
2. 如果走了错误的策略组，检查规则集是否覆盖了该域名
3. 如果策略组正确但节点不通，尝试切换地区

## 注意事项
1. **安全**：`configs/v4.yaml` 在本地填写后会包含机场订阅 URL 和住宅 IP 凭据，请勿把含真实密钥的文件推送到公开远程仓库。建议定期到机场后台重置订阅 Token。
2. **TUN 模式**：当前 `auto-route/auto-redirect/auto-detect-interface` 均为 `true`。如果与 OpenWrt 插件冲突，将这三项改为 `false`（插件会自行管理路由表）。
3. **AirportC**：如果订阅域名被墙，将其 `proxy: DIRECT` 改为 `proxy: AirportA`，借道已加载的 A 机场去拉取。
4. **规则集更新**：上游仓库可能随时调整文件路径。如果启动报错，检查对应 URL 是否仍返回 HTTP 200。
